Graphics and Multimedia You have a minimum of two regular and two hyperlinked graphics. Text wraps around graphics to avoid awkward white space I have two regular images in my photo gallery and two hyperlinked photos on my prompts page, as well as hyperlinked images to social media sites on each page. There are two regular images on my events page that go with their respective workshops in Calvert County. Images are centered, there are no awkward text around the images.
There are no broken graphics or problems with aspect ratio. All graphics are well optimized for the web There are no problems with aspect ratio because I have sized them correctly in the HTML. The image file formats are optimized for the web. I used PNG, JPG, and GIF formats.
The site includes an original multimedia element that you create (Audio, Video, Flash, HTML5 Canvas, etc). This element should make sense in the context of the site I created an animated GIF seen on the Home page complete with dancing crabs holding carrots and pencils.
Form Your site includes a Web Form: Your form must be functional in order to score points.

Your form includes a minimum of one text input field, one scrolling text box (textarea), a checkbox or radio button, and a select list.

Your form includes fieldset and legend elements

Your form includes label and tabindex accessibility enhancements

Your form must be styled with CSS so that form fields are a color other than the default white.

It must contain at least one required field (marked by an asterisk*)

It must be validated using a JavaScript form validator. You can write this yourself or you can tap an existing script (please cite source). 		I have two forms, one is the sign up form, and contant form. Only the CONTACT form uses the JavaScript form validator, and between the two forms they use all the fields (SIGNUP form uses text input, checkboxes, and radio buttons, CONTACT form uses text input, textarea, and select list). Form backgrounds are BLUE. Form is styled in CSS. The form includes label and tabindex accessibility enhancements. Forms include fieldset and legend elements. They are both functional.

Your Pages validate without errors using the W3C HTML5 Validator and the W3C Validated HTML5 icon is placed at the bottom of the each page (only if page validates without errors) All HTML/CSS validations appear in the FOOTER area of each page. Example page.

Your site includes TWO of the following: A CSS based photo gallery, a working demo of a shopping cart ,or a within-the-site search I have a CSS based photo gallery on my image prompts page. A Google custom search bar appears on every page except the requirements page.
Design and Usability The site demonstrates an understanding of the 4 Page Layout Principles as described in class: Contrast, Proximity, Alignment, and Repetition I have used a base for all of my pages. The background, navigation, and sidebar are the same on each page.
Site navigation is easy to use The navigation bar appears near the top of every single page on the website. There is a hover effect to let people know that there is a link.
The text is of a size, color, and style that is easy to read on the chosen background color or graphic Text color I use is black and dark grey, and the backgrounds I use are white and light blue in color. The text sizes are 16px and above. I use the orange and red colors for my links so that they can pop out on white and blue.
There are unifying elements that make the pages seem that they belong together All pages have a structure that makes them seem that they belong together. The same header, footer navigation bar, and sidebar content (besides this page), appears on the structure of each page of the website.
Text and graphic placement is visually appealing I used normal and large text so that it can be easily read. I used Google fonts. Texts and graphics are placed neatly in areas so that people know what they are reading.
Each page should have meta tags that are appropriate to that page including description, keywords, robots, author and copyright tags. Meta tags placed in the head of each webpage.
